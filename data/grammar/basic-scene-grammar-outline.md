# 基础场景句法库第一版

这份大纲对应正式种子文件 [basic-scene-grammar.seed.json](/e:/codework/words/data/grammar/basic-scene-grammar.seed.json)，目标不是做一本“按词义背单词”的词典，而是做一套“什么场景更适合什么结构”的学习卡系统。

## 主卡范围

第一版先按 `60` 张主卡设计，分成 8 组：

- 原因与结果：8 张
- 让步与对比：7 张
- 条件：6 张
- 时间与顺序：6 张
- 名词性从句：7 张
- 定语从句：8 张
- 目的、比较与替代：7 张
- 功能句型与强调结构：11 张

## 为什么这样分

- 这些结构都足够高频，适合日常造句。
- 它们之间有明显对比关系，适合做“场景判断”训练。
- 多数卡都能扩展出重写题、套写题和自由造句题。
- 词汇学习可以自然挂靠在结构卡上，不必把所有基础词都独立成卡。

## 适合独立成卡的内容

适合独立成卡的通常有这几类：

- 后面接法不同：`because + clause` / `because of + noun`
- 语气差异明显：`because` / `since` / `as`
- 场景差异明显：`although` / `even though` / `despite`
- 句法功能不同：`that-clause` / `what-clause` / `whether-clause`
- 能形成固定骨架：`the reason is that...` / `it's + adj + to...`
- 容易写错：`unless` / `in case` / `relative clauses`

## 只做附属词或附属表达的内容

下面这些不一定都要独立成卡，更适合作为主卡里的对比词、搭配词、补充模板：

- 因果连接词：`therefore`, `thus`, `so`, `that's why`
- 转折补充词：`but`, `yet`, `still`, `though`
- 条件补充词：`provided that`, `on condition that`
- 时间补充词：`as soon as`, `by the time`
- 结果骨架：`so...that...`, `such...that...`
- 偏口语邀请表达：`why don't we...`, `do you want to...`
- 替代表达：`instead`, `would rather`
- 困难表达：`have difficulty doing`, `can't manage to...`

## 如果要压缩到 50 张左右

如果你想把第一版先压到 `50` 张左右，建议把下面这些先降为附属内容：

- `due to + noun`
- `in spite of + noun / v-ing`
- `while + clause (contrast)`
- `in case + clause`
- `when relative clause`
- `why relative clause`
- `there is / there are`
- `have trouble + v-ing`
- `it is / was ... that ...`
- `thanks to + noun`

这样保留最核心的高频结构，仍然足够支撑第一版上线和造句训练。

## 二期可替换候选

如果你后面更想靠近“语法教材型”结构库，而不是更偏“场景表达型”结构库，可以优先考虑把下面这些替换进主卡：

- `so + adj / adv + that + clause`
- `such + noun + that + clause`
- `though + clause`
- `whereas + clause`
- `no matter wh- + clause`
- `provided / providing that + clause`
- `only if + clause`
- `as soon as + clause`
- `by the time + clause`
- `it takes + time + to...`
- `comparative + than`
- `verb + v-ing` 这类动名词宾语结构

这些候选更偏“系统语法覆盖”，适合在第一版跑通之后，再根据用户练习反馈替换或扩容。

## 推荐推进顺序

- 第一步：先用前 24 张 MVP 主卡跑通学习、评测和历史链路
- 第二步：扩到 50-60 张正式主卡
- 第三步：再把附属词、搭配词、骨架表达补到每张卡里
- 第四步：根据用户常错项决定哪些附属表达升级为独立主卡
